리눅스의 top, ps, jobs, kill 명령어에 대한 조사와 정리

top
- 현재 OS의 상태를 나타내주는 CLI 어플리케이션

1. 요약 영역
   - top의 상단에 위치해 있다. 전체 프로제스가 OS에 대해서 리소스를 어느정도 차지하는지 알려줌
   - 시스템의 시간, OS가 살아있는 시간, 유저 세션수, 로드 애버리지, Tasks, cpu-메모리 사용량을 표시

 2. 디테일 영역
    - 각 프로세스에 대하 자세한 내용을 보여줌
    - pid(프로세스 id), user(해당 프로세스를 실행한 user 또는 효과를 받는 user), pr(커널에 의한 스케줄링 우선순위), ni(pr에 영향을 주는 nice의 값), virt(프로세스가 소비하는 총 메모리), res(ram에서 사용중인 메모리의 크기), shr(다른 프로세스와의 공유 메모리), s(프로세스의 현재 상태), %cpu, %mem(rem에서 res가 차지하는 비율), time+(프로세스가 사용한 토탈 cpu 시간), command(해당 프로세스를 실행한 커맨드 표시)를 표시

ps
- process status의 줄임말, 현재 구동 중인 프로세스 정보를 확인 가능
- 주요 옵션은 -e, -f, -l이 존재
- -e: 현재 사용자 뿐아니라 다른 사용자들이 구동 시킨 모든 프로세스를 보여줌
- -f: 보다 상세한 정보를 보여줌
- -l: -f보다 더 자세한 정보를 보여줌
  - f(프로세스 플래그), s(프로세스의 현재 상태), uid(프로세스를 실행시킨 사용자 id), pid(프로세스에 부여된 id), ppid(프로세의 부모 프로세스 id), c(cpu 사용량(%)), pri(프로세스 우선 순위), ni(프로세스의 cpu 자원 사용 우선순위), addr(프로세스의 메모리 주소), sz(가상 메모리 사용량), stime(프로세스 시작 시간), tty(프로세스가 실행된 터미널의 종류와 번호), time(프로세스에 의해 사용된 cpu 시간), cmd(실행된 프로세스의 이름 혹은 실행된 명령)를 표시

jobs
- 현재 쉘 세션에서 실행 중인 프로세스 목록을 보여줌
- 쉘 세션에서 실행 중인 작업 목록을 간단히 확인 가능하지만, 자세한 정보는 보여주지 않으며 백그라운드에서 실행 중인 다른 쉘 세션의 작업 목록을 확인할 수 없다.

kill
- 프로세스에 시그널을 보내 원하는 동작을 수행 프로세스 종료를 위해 사용함
- kill 명령어를 사용하기 위해서는 프로세스 id를 알아야함
- kill의 명령어 옵션
  - -s(특정 시그널을 사용해 프로세스를 종료), -l, --list(지원되는 시그널 목록을 출력), -a, --all(현재 사용자에 속한 모든 프로세스를 종료), -q, --queue(프로세스에 시그널을 보내는 대신 스그널을 대기열에 추가)
- 간단한 명령어로 프로세스를 종료 가능하나 의도치않은 프로세스 종료가 발생할 수 있음
